MyServletContextListener - класс прослушивает 2 ключевых события:
1. Инициализация контекста сервлета (ServletContext)
2. Уничтожение контекста
Мы получаем событие во время инициализации контекста:
 - получаем параметры инициализации контекста
 - используем полученные параметры (например для соединения с базой данных), в нашем случае используем кличку для собаки
 - используем полученный объект как атрибут контекста, так что все части приложения имеют к нему доступ

Существуют 3 области видимости атрибутов: Context, Request, Session
Context Attributes - любая часть приложения имеет доступ к атрибутам контекста (ServletContext)
Session Attributes - доступ есть только у того, кто имеет доступ к HttpSession (ServletRequest)
Request Attributes - доступ есть только у того, кто имеет доступ к ServletRequest (HttpSession)


Для того что безопасную однопоточную модель SingleThreadModel необходимо, чтобы класс
наследовался от HttpServlet и реализовывал интерфейс SingleThreadModel - такой сервлет сможет
обрабатывать только один запрос(request) за раз. Такой механизм поможет защитить только
перемнные экземпляров класса, но не поможет с переменными самого класса. Но вообще лучше избегать
данного метода (deprecated)

Потокобезопасными являются только Request атрибуты и локальные переменные
