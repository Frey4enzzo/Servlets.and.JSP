Context Attributes:
1. Not thread-safe
2. Уровень доступа (кто видит): любая часть приложения (сервлеты, JSP страницы, слушатели)
3. Область/Scope (время жизни): равна времени жизни ServletContext, т.е. времени жизни работы
задеплоенного приложения.
4. Для чего использовать: ресурсы, которые необходимы приложению (подключение к базе данных, email админа, и.т.д.)

HttpSession Attributes:
1. Not thread-safe
2. Уровень доступа (кто видит): любой сервлет или JSP с доступом к этой особой сессии. Сессия связывает множественные
запросы с одним и тем же пользователем, который использует разные сервлеты.
3. Область/Scope (время жизни): время жизни сессии (сессия может быть уничтожена программно или по таймауту)
4. Для чего использовать: данные и ресурсы связанные с клиентской сессией, а не только с одиночным запросом.
Например интернет корзина для покупок

Request Attributes
1. Thread-safe
2. Уровень доступа (кто видит): любая часть приложения, которая имеет прямой доступ к объекту request
3. Область/Scope (время жизни): время жизни запроса(request), т.е. время выполнения service() метода (doGet(), doPost())
4. Для чего использовать: передача информации о модели с контроллера на представление


ПАРАМЕТРЫ ИНИЦИАЛИЗАЦИИ <init-param> используются только для инициализации параметров сервелата <servlet-param>
